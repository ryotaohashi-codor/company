## 1. HP制作の進め方

### フェーズ1: ヒアリング・要件定義（1-2日）

**確認すべき項目**:
- ページの目的（訪問者にどう行動してほしいか）
- ターゲット訪問者（属性・デバイス）
- 競合HP・参考サイト
- 既存コンテンツの活用有無
- 公開期限

**成果物**: 要件定義書、ワイヤーフレーム案

### フェーズ2: デザイン・ワイヤーフレーム（3-5日）

**チェックポイント**:
- モバイル対応を最優先（mobile-first設計）
- ナビゲーションは3クリック以内で主要ページへ到達可能
- CTA（行動喚起ボタン）を目立つ位置に配置
- 読みやすいフォントサイズ（本文16px以上）

**デリバラブル**: HTML/CSSベースのプロトタイプ、スクリーンショット

### フェーズ3: コンテンツ作成（5-10日）

**内容作成のコツ**:
- 訪問者の行動フローに沿ってコンテンツを順序付け
- 各セクションに「これを読むと何がわかるのか」を明示
- 画像・動画は定期的に更新しやすい構成に
- リンクは同じウィンドウ/新規タブを意識的に使い分け

**SEO対策（初期設定）**:
- ページタイトル: キーワード含む、40字以内
- メタディスクリプション: 120字以内
- h1タグは1ページに1つだけ

### フェーズ4: テスト・公開（2-3日）

**テストチェックリスト**:
- [ ] 全リンクが動作するか（外部リンク含む）
- [ ] 画像が正しく表示されるか
- [ ] モバイル・タブレット・PCで表示確認
- [ ] 主要ブラウザ（Chrome, Safari, Edge）での動作確認
- [ ] ページ読み込み速度（目標: 3秒以内）

**公開時の確認**:
- DNS設定・SSL証明書確認
- Google Search Consoleへの登録
- アナリティクス計測コード設定

## 2. ページレイアウト・デザインガイド

### よくあるレイアウトパターン

| パターン | 用途 | 注意点 |
|---------|------|--------|
| **1カラム（全幅）** | LP・ランディング | スクロール量を意識 |
| **2カラム（メイン+サイド）** | ブログ・情報サイト | サイドバー幅は1/4程度 |
| **3カラム** | ポータルサイト | 情報過多になりやすい |

### レスポンシブ設計（必須）

```
- PC: 1200px以上
- タブレット: 768px-1200px
- スマートフォン: 375px-768px
```

**チェック項目**:
- [ ] ブレークポイントで適切にレイアウト変更
- [ ] タッチターゲット（ボタン）が44px以上
- [ ] 横スクロールが発生しない

## 3. SEO・パフォーマンス最適化

### ページスピード改善

**優先度高い施策**:

1. **画像最適化**
   - JPG/PNG → WebP形式に変換
   - 各デバイスに合わせた画像サイズ出し分け
   - Lazy Loading実装

2. **CSS/JavaScript削減**
   - 不使用コードの削除
   - CSSの圧縮・ミニファイ
   - JavaScriptは非同期読み込み（async）

3. **キャッシュ設定**
   - 静的ファイル（画像・CSS）: 1ヶ月以上
   - HTMLファイル: 短めに（1-7日）

### SEO施策の優先順位

**Phase 1（初期）**:
- ページタイトル・メタディスクリプション最適化
- h1-h3の階層構造を正しく
- 内部リンク構造の最適化

**Phase 2（1-3ヶ月）**:
- Googleビジネスプロフィール登録（店舗ある場合）
- サイトマップ生成・送信
- モバイル対応確認

**Phase 3（継続）**:
- 検索流入キーワード分析
- コンテンツ更新・追加
- バックリンク獲得

## 4. ユーザビリティチェックリスト

| 項目 | チェック方法 | 基準 |
|------|----------|------|
| **ナビゲーション** | 目的ページに何クリックで到達? | 3クリック以内 |
| **読みやすさ** | フォントサイズ・行間 | 本文16px、行間1.5倍 |
| **CTAの明確性** | 何をすべきか一瞬で分かるか | 分かる |
| **連絡先の探しやすさ** | 「お問い合わせ」は何クリックで? | 1-2クリック |
| **エラー処理** | 404ページは分かりやすいか | 代替案を提示 |

## 5. アナリティクス設定・運用

### 計測する主要指標（Google Analytics 4）

- **ページビュー**: 各ページの訪問数
- **平均エンゲージメント時間**: ページの滞在時間
- **離脱率**: ページから離脱したユーザー割合
- **コンバージョン**: 問い合わせ・購入などの目的達成

### 月次レビューのポイント

毎月、以下の観点で分析：

- 「どのページへのアクセスが多いか」 → コンテンツ指針
- 「どこから流入しているか」 → チャネル改善
- 「どこで離脱しているか」 → ボトルネック改善

## 6. よくある落とし穴と対策

| 失敗 | 原因 | 対策 |
|------|------|------|
| **スマホで崩れる** | 設計時にPCのみ確認 | mobile-first設計を実践 |
| **重すぎてやたら遅い** | 画像を圧縮せず使用 | 全画像をWebP化 |
| **どこにお問い合わせがあるか分からない** | CTAの配置が不適切 | ファーストビューに1つ、フッターに1つ |
| **SEOで全く見つからない** | タイトル・メタディスが不適切 | キーワード調査→タイトル設定 |
