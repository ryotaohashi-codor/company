## 1. 講座企画・トピック選定

### トピック選定のプロセス（1-2週間）

**ステップ1: 市場ニーズの調査**

- Udemyのカテゴリ別ランキングを確認（売上TOP100）
- Amazonビジネス書でキーワード検索（月間検索数）
- Google トレンドで検索トレンド確認（右肩上がりか）

**判断基準**:
- 競合講座が10-50件（需要あり、かつ参入余地あり）
- 月間検索ボリューム: 100以上
- トレンド: 安定または上昇傾向

**ステップ2: 自分たちの専門性の確認**

- 講座を教えられるレベルか
- 専門知識に自信があるか
- サンプル動画を作れるか

**ステップ3: ターゲット学習者の定義**

```
対象者: [職業/レベル]
例: 「営業未経験の新入社員」「Python初心者」
年齢層: [年代]
課題: [何に困っているのか]
期待: [学習後に何ができるようになりたいのか]
```

### トピック決定のチェックリスト

- [ ] 競合講座分析が完了（TOP5講座を研究）
- [ ] 市場需要が確認されている
- [ ] ターゲット学習者が明確
- [ ] 自分たちが教える自信がある
- [ ] 3-6ヶ月で完成できる範囲か

### 参考資料の活用

新規講座企画時に、`projects/reference/subtitle-library/` から過去の完成講座を参照してください：

**参照ポイント**:
- **構成**: セクション分割、レッスン数の目安
- **スタイル**: 説明の詳しさ、事例の挙げ方、用語の説明方法
- **長さ**: 各レッスンの標準的な動画時間
- **難度曲線**: 導入から応用までの段階的な進め方

**参照方法**:
```
projects/reference/subtitle-library/
├── 2026-01_データ分析入門/    ← 同じカテゴリの講座を参考に
│   ├── lecture-outline.md      ← 構成パターン
│   └── subtitles/ja.vtt        ← 説明スタイル
└── 2026-05_Python基礎/
    └── ...
```

## 2. カリキュラム設計

### セクション・レッスン構成の考え方

**原則**: 「学習者のゴール達成」を軸に、逆向き設計

```
最終ゴール
  ↓ (何ができれば達成?)
実装スキル群
  ↓ (何を知る必要がある?)
基礎知識
  ↓ (前提条件は?)
導入・概要
```

**例: 「Python入門」の場合**

```
【最終ゴール】 Pythonで簡単な自動化スクリプトが書ける

【実装スキル】 変数・関数・制御構造の使用、ライブラリの活用

【基礎知識】 プログラミングの基本概念、Pythonの特性

【導入】 Pythonとは何か、何ができるのか、環境構築
```

### セクション数・レッスン数の目安

**受講者が続けやすい構成**:
- セクション数: 6-12（多すぎると挫折）
- 1セクションのレッスン数: 5-8個
- 1レッスンの長さ: 5-15分（動画）

**講座全体の時間**:
- 初級: 5-10時間
- 中級: 10-20時間
- 上級: 20-40時間

### カリキュラムのテンプレート

```markdown
## セクション1: 導入・全体像
- レッスン1: このコースで学ぶこと（2分）
- レッスン2: [基本概念]（5分）
- レッスン3: [用語解説]（3分）

## セクション2: 基礎
- レッスン4: [概念1の詳細]（8分）
- レッスン5: [実践演習1]（10分）
- ...

## 最終セクション: 総まとめ
- 修了プロジェクト（学習者が実装）
```

## 3. 講座ページの最適化（重要）

### タイトル・説明文の工夫

**講座タイトル（60字以内）**

```
❌ 「Pythonプログラミング」（競合多い、何ができるか不明）
✅ 「20時間で完全習得！初心者向けPython実務講座」（ターゲット・成果明確）
```

**学習目標の表示（箇条書き 5-8個）**

```
このコースを受講することで、以下ができるようになります：
- Python環境のセットアップ
- 変数・データ型の使用
- 関数の作成と利用
- ライブラリの活用
- 実務的なスクリプト作成
```

**講座説明文（150-200字）**

```
[誰向け]Pythonをこれから学ぼうとしている初心者向けです。

[何ができるか]本コースを完了すると、Pythonで実務的なスクリプトが
自分で書けるようになります。

[何が特徴か]基礎から実践まで、実装を通じて学べます。毎レッスン
に演習がついており、すぐに手を動かせます。

[前提知識]プログラミング経験は不要です。
```

### サムネイル・プレビュー画像

**チェックポイント**:
- [ ] テキストは大きく、遠くからでも読める
- [ ] 背景色とテキストの対比が高い
- [ ] 競合講座と並べたときに目立つか
- [ ] 縦横比が正しい（1920x1080px推奨）

## 4. 動画コンテンツ制作

### 動画の品質基準

**音声**:
- クリアな音声（バックノイズなし）
- 適切なボリューム（喋り声と効果音のバランス）
- 話すスピード: 通常より遅め（理解しやすく）

**画面**:
- フォントサイズ: 本文は24pt以上（視聴者が読みやすく）
- 背景: シンプル（コードやスライドに集中）
- 動画品質: 1080p以上（高品質）

### 1レッスン動画の構成（例：10分動画）

```
オープニング（30秒）
  ├─ このレッスンで学ぶこと
  └─ 学習目標

本体（8分）
  ├─ 概念説明（3分）
  ├─ デモ・実装（4分）
  └─ ポイント解説（1分）

クロージング（1.5分）
  ├─ まとめ
  ├─ 次レッスンへの橋渡し
  └─ 質問・困ったことの案内
```

## 5. 受講者サポート・Q&A対応

### フォーラム・Q&A管理

**回答時間の目安**: 24時間以内

**よくある質問の対応**:
- 「環境構築でエラーが出た」→ OS別・バージョン別の対処法を用意
- 「このレッスンの後、どう進めればいい?」→ 次のステップを明示
- 「もっと詳しく知りたい」→ 参考リソースを紹介

### 受講生からのフィードバック活用

**毎月チェック**:
- 「どのレッスンが理解しづらいか」 → カリキュラム改善
- 「どんな質問が多いか」 → コンテンツ追加検討
- 「満足度スコア」 → レビュー改善のヒント

## 6. 価格設定・プロモーション

### 初期価格設定

**参考: Udemy市場価格（カテゴリ別）**

| カテゴリ | 初級 | 中級 | 上級 |
|---------|------|------|------|
| **プログラミング** | $15-25 | $25-50 | $50-100 |
| **ビジネススキル** | $15-25 | $20-40 | $35-75 |
| **デザイン** | $15-25 | $25-45 | $45-80 |

**設定のコツ**:
1. 競合TOP5の価格を確認
2. 自分たちの講座の充実度と比較
3. 安すぎない（品質の信号）、高すぎない（参入障壁）のバランス

### プロモーション施策

**リリース直後（最初の2週間）**:
- 限定割引クーポン（30-50%OFF）を友人・ネットワークに配布
- 受講者増 → 高評価獲得 → アルゴリズム有利化

**継続的**:
- 季節セール（Udemyの大型セールに合わせる）
- クーポン配布（SNS・ブログで告知）
- コンテンツ更新（新レッスン追加で検索ランク向上）

## 7. レビュー・評価向上施策

### 高評価を得るためのポイント

**講座品質**:
- レッスン完成度 → 何度も見直し、改善
- サポート体制 → 質問に24時間以内で回答
- 内容の更新 → 最新情報を反映

**受講者体験**:
- わかりやすさ → 初心者目線で説明
- 実践性 → すぐに使えるスキルを教える
- 達成感 → ゴール達成を明確にする

### 低評価コメントへの対応

```
「難しすぎてついていけなかった」
→ 「初級レッスンを追加予定です。フォーラムで相談ください。」

「動画の音が聞き取りにくい」
→ 「申し訳ありません。再録音して更新いたします。」
```
