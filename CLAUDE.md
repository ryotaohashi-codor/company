# 司令塔：Claude Code エージェント・システム

**役割**: 4つのビジネス機能を専門家チームで実行する統括指揮所

---

## システム構成

- **CLAUDE.md** (このファイル): ルーティング判断
- **agents/**: 4つの専門部門エージェント
- **.claude/commands/**: 各部門のコマンド定義
- **guidelines/**: 業務マニュアル
- **templates/**: 出力テンプレート

---

## ルーティング判断表

| 指示内容 | 対応部門 | 主な役割 |
|---------|--------|--------|
| HP企画・設計・制作・公開・保守 | **ウェブサイト開発** | サイト構成設計、UI/UX、技術実装、SEO |
| 講座企画・カリキュラム・教材制作・公開 | **オンライン講座開発** | 講座設計、教材制作、学習体験、成果測定 |

| 講座の鮮度チェック・品質診断・メンテナンス判定 | **講座メンテナンス・品質管理** | 更新判定、優先度提案、品質監査 |

---

## 実行パターン

### パターンA: 単一部門タスク
```
指示 → 判定 → 該当部門を起動 → 実行 → 報告
```

**例**:
- 「HPの [ページ] を作成してください」→ ウェブサイト開発部門
- 「Udemy講座の [内容] を企画してください」→ オンライン講座開発部門
- 「コミュニティで [質問] への回答を用意してください」→ コミュニティ運営部門

### パターンB: 複合タスク（複数部門）
```
指示 → 判定 → 複数部門を並列起動
     ↓
  各部門が独立実行（ファイル成果物を出力）
     ↓
統合部門が結果を集約 → 最終報告
```

**例**:
- 「新しい講座を企画し、HPにコース紹介ページを作成してください」
  - オンライン講座開発部門: 講座企画書作成
  - ウェブサイト開発部門: コース紹介ページ制作
  - コミュニティ運営部門: コミュニティ内告知準備

## 使用方法

### 基本形式

```
【指示内容】
[具体的なタスク]

【背景・文脈】（必要に応じて）
[なぜ必要か、制約条件など]
```

### 複合タスク時

```
【メインタスク】[主導部門]対象
[内容]

【並列実行項目】
- [副部門1対象の連携タスク]
- [副部門2対象の連携タスク]
```

## デザイン方針

1. **シンプルさ**: 4つの部門に特化した焦点化した構成
2. **実行性**: すぐに使える実務的な内容
3. **成長性**: 運用しながら改善する前提

---

**このシステムで実現すること**:
- HPの企画から公開まで、1つの指示で完結
- Udemy講座の企画から制作、運営、メンテナンスまで一貫対応
- コミュニティ管理と品質管理の統合
- 複数タスクは並列実行で効率化
- 講座の長期的な品質と鮮度を継続的に監視

最後に、プロジェクト達成に必要なすべてのドキュメントが一箇所に集約されています。
